# ☕ Cafe Markul: Horror Shift in the Coffeehouse

**Cafe Markul** — это атмосферная хоррор-игра от первого лица, созданная на **C# WinForms**.  
Игрок проходит смену в небольшой кофейне: включает свет, обслуживает посетителей, готовит напитки, взаимодействует с предметами и постепенно сталкивается с тревожной атмосферой, которая скрывается за обычной рабочей сменой.

Проект сочетает симулятор работы бариста, сюжетные диалоги, ручную псевдо-3D-графику, звуковое сопровождение, автосохранение и хоррор-заставки.

---

## 🎮 Игровой процесс

В начале игры игрок попадает в стартовое меню, после чего проходит Loading-заставку и начинает **День 1** в кофейне.

Основная задача игрока — выполнять цели по смене:

| Действие | Описание |
|---|---|
| 🚪 Начать игру | Запуск новой игры с заставки и первого дня |
| 💾 Продолжить игру | Восстановление с автосохранения |
| 💡 Включить / выключить свет | Взаимодействие с выключателем |
| 📺 Включить / выключить телевизор | Телевизор влияет на музыку и цели |
| ☕ Готовить напитки | Использование кофемашины, молока, сиропа и стаканов |
| 🧊 Добавлять ингредиенты | Кофе, молоко, сироп, лёд |
| 🧼 Мыть чашки | Задание с раковиной и звуком мойки |
| 👥 Общаться с NPC | Диалоги с Тией и Майком |
| 💵 Получать оплату | Деньги появляются после выдачи заказа |
| 🌙 Завершить смену | Выключить телевизор, затем свет |

Игрок перемещается по кофейне от первого лица, смотрит на предметы и взаимодействует с ними, если находится рядом с объектом и направлен в его сторону.

## 🧩 Основные механики

### 🎯 Система целей

В игре есть список текущих заданий.  
Цели постепенно направляют игрока по смене: от первых действий в кофейне до обслуживания клиентов и завершения дня.

Примеры целей:

- включить свет;
- включить телевизор;
- собрать грязные кружки;
- вымыть чашки;
- приготовить заказ;
- отдать заказ клиенту;
- выключить телевизор;
- выключить свет.

После выполнения одной цели открывается следующая. Благодаря этому игрок понимает, что нужно делать дальше.

---

### 👀 Интерактивные зоны

Каждый важный предмет имеет свою зону взаимодействия:

| Объект | Что делает игрок |
|---|---|
| Выключатель | Включает и выключает свет |
| Телевизор | Включает экран и музыку / выключает телевизор |
| Холодильник | Открывает дверцу и берёт молоко |
| Кофемашина | Готовит кофе |
| Раковина | Запускает мойку кружек |
| Касса | Показывает рецепт заказа |
| Полки | Позволяют брать стаканы |
| Клиенты | Запускают диалоги и выдачу заказа |

Игрок должен не только находиться рядом с объектом, но и смотреть примерно в его сторону. Это делает взаимодействие более похожим на игру от первого лица.

---

### ☕ Приготовление напитков

В игре есть заказы посетителей.  
Игрок должен правильно использовать предметы кофейни:

- взять нужный стакан;
- добавить кофе;
- добавить молоко;
- добавить сироп;
- добавить лёд;
- проверить рецепт на кассе;
- выдать готовый напиток клиенту.

Состав напитка хранится в логике игры, поэтому заказ нельзя выдать неправильно: игра проверяет, достаточно ли ингредиентов.

---

### 👥 NPC и диалоги

В игре есть персонажи:

| Персонаж | Роль |
|---|---|
| Тиа | Посетительница, делает заказ и участвует в диалоге |
| Майк | Посетитель, связанный с важной сюжетной частью |

NPC не просто стоят на месте. Они могут:

- появляться в сцене;
- идти к стойке;
- разговаривать;
- ждать заказ;
- держать предметы;
- получать напиток;
- уходить;
- менять эмоции.

У Майка есть отдельные состояния: широкие глаза, улыбка, движение, прощальная реплика и музыкальный момент.

### 💾 Автосохранение

В игре реализовано автосохранение в контрольной точке.  
Если игрок дошёл до нужного момента, при следующем запуске появляется меню:

| Кнопка | Назначение |
|---|---|
| Продолжить игру | Восстановить игру с автосохранения |
| Начать игру заново | Удалить старое сохранение и начать с начала |

При продолжении восстанавливаются:

- стадия сюжета;
- текущие цели;
- положение камеры;
- состояние телевизора;
- включённая музыка;
- прогресс игрока.

---

### 🎬 Заставки

В игре есть несколько заставок:

| Заставка | Описание |
|---|---|
| Логотип Cafe Markul | Плавное появление названия игры |
| Loading | Хоррор-сцена с девушкой, шагами и нападающим |
| День 1 | Переход перед началом первого дня |
| День 2 | Появляется после выключения света в конце смены |
| Продолжение следует | Финальный экран без ограничения по времени |

Loading-заставка сделана вручную через 2D-графику: персонажи состоят из фигур, а шаги и звук смерти синхронизированы по времени.

## 🧠 Архитектура

Проект построен по принципу разделения ответственности.

### Model

Модель хранит данные сцены:

- точки;
- рёбра;
- грани;
- коллайдеры;
- камеру.

Модель не отвечает за сюжет и не рисует игру. Она хранит геометрию и состояние сцены.

### View

Представление отвечает за отрисовку:

- 3D-сцены;
- интерфейса;
- заставок;
- Loading-анимации;
- текста;
- рецептов;
- эффектов.

### Controller

Контроллеры управляют логикой:

- `MainController` управляет сценой и камерой;
- `HorrorGameController` управляет сюжетом и задачами;
- `AudioManager` управляет звуком;
- `Form1` связывает ввод, отрисовку и игровые системы.

Такой подход помогает разделить код на части: одна часть хранит данные, другая рисует, третья управляет поведением игры.

## 📐 Алгоритмы и технические решения

### 1. Собственная псевдо-3D-сцена

Сцена собирается вручную из:

- `Point3D` — точек в пространстве;
- `Edge` — рёбер;
- `Face` — граней;
- `BoxCollider` — зон столкновения.

Каждый объект кофейни состоит из координат и поверхностей.  
Это позволяет создать комнату, мебель, холодильник, стойку, полки, телевизор и другие предметы без игрового движка.

---

### 2. Поворот точки вокруг оси Y

Для работы с 3D-координатами используется формула поворота:

```csharp
double cos = Math.Cos(angle);
double sin = Math.Sin(angle);

double oldX = X;
double oldZ = Z;

X = oldX * cos + oldZ * sin;
Z = -oldX * sin + oldZ * cos;
```

Этот метод нужен для поворота объектов в пространстве.

---

### 3. Камера от первого лица

Камера имеет координаты:

- `X`;
- `Y`;
- `Z`;
- `Yaw`;
- `Pitch`.

Мышь управляет поворотом камеры.  
Каждый кадр игра смотрит, насколько мышь отклонилась от центра экрана, поворачивает камеру и возвращает курсор обратно в центр.

---

### 4. Коллизии

Для стен и мебели используются прямоугольные коллайдеры.  
Перед перемещением игрока проверяется, не пересекается ли он с объектами.

Это не даёт игроку проходить сквозь:

- стены;
- барную стойку;
- мебель;
- холодильник;
- столы;
- другие препятствия.

---

### 5. Проверка направления взгляда

Интерактивный предмет доступен только тогда, когда игрок:

1. стоит рядом;
2. смотрит на предмет.

Для этого используется направление камеры и направление от игрока к объекту.  
Если угол подходит, появляется подсказка взаимодействия.

---

### 6. Игровой цикл

Главный цикл находится в `GameTimer_Tick`.

Каждый тик:

- считается `deltaTime`;
- обновляется ввод;
- двигается камера;
- обновляется логика игры;
- обновляются NPC;
- проверяются цели;
- проигрываются звуки;
- перерисовывается экран.

---

### 7. Анимации по времени

Многие эффекты завязаны на таймер:

- Loading-сцена;
- шаги девушки;
- звук смерти;
- движение клиентов;
- открытие холодильника;
- приготовление кофе;
- мойка кружек;
- переходы дня;
- финальный экран.

Благодаря этому игра не зависит напрямую от скорости компьютера, а использует прошедшее время.

## 🔊 Звуковая система

За звук отвечает `AudioManager`.

В игре есть:

| Тип звука | Пример |
|---|---|
| Короткий звук | шаг, дверь, лампа |
| Звук с ограничением | отдельные действия |
| Loop-звук | музыка, мойка кружек |
| Фоновая музыка | телевизор / игровые моменты |

`AudioManager` умеет:

- запускать музыку;
- останавливать музыку;
- запускать несколько коротких звуков подряд;
- регулировать громкость;
- поддерживать loop-звуки;
- освобождать ресурсы при закрытии игры.

## ⚙️ Сборка и запуск

### Требования

Для запуска проекта нужны:

- Windows;
- Visual Studio;
- .NET Framework / .NET для Windows Forms;
- подключённая библиотека NAudio;
- папка `Assets/Music` со звуковыми файлами.

### Как запустить

1. Открыть проект в Visual Studio.
2. Убедиться, что все `.cs` файлы находятся в проекте.
3. Проверить, что папка `Assets/Music` подключена и содержит mp3-файлы.
4. Собрать проект через `Build`.
5. Запустить через `Start`.
